Cameron Gutman and Andrew Mason

What's an emulator?

Hardware or software that enables one computer system (host) to behave like another computer system (guest).

  • Our host: MatLab
  • Our guest: MIPS

MIPS

  • Instruction Set Architecture (ISA)
  • Low-level assembly instructions
    • "move this value to this register"
    • "store this byte at this address"

Emulation Steps

  1. Read assembled binary of machine instructions.
    • Parse into op code and arguments.
  2. Determine what instruction is trying to be executed.
  3. Invoke our emulation of the instruction instead.
  4. Update CPU state
    • memory pages
    • register values
    • program counter
    • etc
